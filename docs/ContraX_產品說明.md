## 1. 產品簡介

### 1.1 產品概述

「ContraX 合約管理系統」是一套專為中小企業設計的單機版桌面應用程式，用於管理職人（作家、編劇、導演、漫畫家、藝人等）與各合作對象之間的合約。系統提供完整的合約生命週期管理，從合約建立、進度追蹤、收款提醒到完成歸檔，一站式解決合約管理需求。

### 1.2 目標使用者

本系統適用於：

- 中小企業行政人員
- 合約管理專員
- 財務收款人員
- 公司主管（查詢與報表）

### 1.3 系統特色

| 特色 | 說明 |
|------|------|
| 跨平台支援 | 同時支援 macOS 與 Windows 作業系統 |
| 單機運作 | 無需網路連線，資料存放於本機，安全可靠 |
| 自動編號 | 依類別自動產生合約流水號，避免人工錯誤 |
| 主動提醒 | 收款與發票開立時程自動提醒，不漏任何款項 |
| Google 日曆整合 | 同步排程至 Google Calendar，5 天前自動提醒 |
| 調閱合約 | 上傳、預覽、下載合約文件，支援版本管理 |
| 快速查詢 | 多條件組合搜尋，秒速找到目標合約 |
| 資料匯出 | 支援匯出 Excel 報表，便於分析與存檔 |

## 2. 功能說明

### 2.1 合約編號自動產生

系統採用三段式編碼規則，自動產生唯一的合約編號：

| 段落 | 格式 | 範例 |
|------|------|------|
| 職人代碼 | 類別 + 三碼編號 | A010（A類作家第010號） |
| 合作類別 | 大類 + 中類 | A01（出版刊載-實體出版） |
| 日期流水號 | YYYYMM + 三碼流水 | 202508001 |

**完整編號範例**：`A010-A01-202508001`

### 2.2 合約基本資料維護

完整記錄合約所需資訊，包含：

- **職人資訊**：類別、代碼、姓名/筆名
- **合作對象**：公司名稱、聯絡人、聯絡方式
- **合約內容**：名稱、年限、版本日期
- **備註說明**：特殊條款、注意事項

### 2.3 合約進度追蹤

系統定義七種標準狀態，清楚掌握每份合約的進度：

| 狀態 | 中文名稱 | 說明 |
|------|----------|------|
| DRAFT | 草擬中 | 合約條文撰寫中 |
| REVIEW | 審核中 | 內部審核確認 |
| NEGOTIATE | 協商中 | 與對方協商條款 |
| SIGNING | 用印中 | 等待雙方用印 |
| ACTIVE | 履約中 | 合約生效執行中 |
| COMPLETED | 已完成 | 合約履行完畢 |
| TERMINATED | 已終止 | 提前終止 |

### 2.4 收款與發票提醒

系統提供主動式提醒機制，確保不遺漏任何收款與發票開立時程：

- 每份合約可設定最多三期收款排程
- 系統啟動時自動檢查即將到期的款項
- 提前天數可自訂（預設 7 天）
- 區分「即將到期」與「已逾期」項目
- 彈出視窗 + 清單顯示雙重提醒

#### Google Calendar 整合

系統支援將收款與發票排程同步至 Google Calendar，透過 Google 的通知機制提供額外提醒：

| 項目 | 日曆事件標題 | 提醒時間 |
|------|-------------|---------|
| 收款提醒 | [ContraX] 收款：{合約名稱} | 到期日前 5 天 |
| 發票開立 | [ContraX] 發票：{合約名稱} | 開立日前 5 天 |

首次使用需進行 Google 帳號授權，之後系統會自動同步排程至您的 Google 日曆。

### 2.5 收款完成確認

收款完成後，系統記錄完整的收款資訊：

- 實際收款日期與金額
- 發票號碼與開立日期
- 差異說明（如金額不符時的備註）
- 自動更新提醒狀態為「已完成」

### 2.6 合約查詢

提供多條件組合查詢，快速找到目標合約：

| 查詢條件 | 說明 |
|---------|------|
| 合約編號 | 支援模糊查詢與精確查詢 |
| 職人名稱 | 依職人篩選所有相關合約 |
| 合作對象 | 依簽約對象公司篩選 |
| 合約名稱 | 關鍵字搜尋 |
| 合約狀態 | 依進度狀態篩選 |
| 日期區間 | 依建立日或到期日篩選 |
| 收款狀態 | 篩選有待收款項的合約 |

### 2.7 調閱合約

系統提供合約文件的上傳、管理與調閱功能，方便快速取得合約原始檔案：

| 功能 | 說明 |
|------|------|
| 上傳文件 | 將合約 PDF、Word 檔或掃描圖檔上傳並關聯至合約 |
| 版本管理 | 同一合約可上傳多個版本，自動記錄上傳時間 |
| 快速預覽 | PDF 與圖片檔可直接在系統內預覽 |
| 下載/開啟 | 將文件下載至本機或使用預設程式開啟 |

**支援格式**：PDF、DOC、DOCX、JPG、PNG、ZIP

## 3. 操作流程

### 3.1 新增合約流程

1. 點選「新增合約」按鈕
2. 選擇職人類別與職人
3. 選擇合作類別（系統自動產生合約編號）
4. 填寫合作對象與合約名稱
5. 設定合約年限與版本日期
6. 新增收款排程（選填）
7. 點選「儲存」完成建立

### 3.2 日常作業流程

系統啟動時的標準作業流程：

1. 啟動系統，檢視待處理提醒
2. 處理逾期或即將到期的收款項目
3. 開立應開發票
4. 更新合約進度狀態
5. 確認收款完成並記錄

## 5. 安裝說明

### 5.1 macOS 安裝

1. 下載 `ContraX-macOS.dmg` 檔案
2. 雙擊開啟 DMG 檔案
3. 將「ContraX.app」拖曳至「應用程式」資料夾
4. 從「應用程式」中啟動 ContraX

### 5.2 Windows 安裝

1. 下載 `ContraX-Windows.zip` 檔案
2. 解壓縮至想要的安裝位置
3. 雙擊「ContraX.exe」啟動
4. （選用）建立桌面捷徑方便日後啟動

## 6. 常見問題 (FAQ)

### Q1: 資料會儲存在哪裡？

資料儲存於本機的使用者資料夾中（SQLite 資料庫），不會上傳至雲端。詳細路徑請參考「4.3 資料儲存位置」。

### Q2: 如何備份資料？

將資料庫檔案 `contracts.db` 複製到外部儲存裝置或雲端空間即可。建議定期備份（每週或每月）。

### Q3: 可以在多台電腦使用嗎？

本系統為單機版，資料不會自動同步。若需在多台電腦使用，可手動複製資料庫檔案進行遷移。

### Q4: 如何匯入現有的 Excel 合約資料？

系統提供 Excel 匯入功能，可將現有的合約清單批次匯入。請確保 Excel 格式符合系統要求的欄位結構。

### Q5: 提醒功能如何運作？

系統提供兩種提醒方式：
1. 系統啟動時自動檢查並顯示提醒清單
2. 同步至 Google Calendar，於事件前 5 天發送 Email 與手機推播通知

### Q6: 如何設定 Google Calendar 同步？

首次使用時，請至「系統設定」開啟 Google Calendar 同步功能，系統會引導您完成 Google 帳號授權。授權完成後，新增或修改收款排程時會自動同步至您的 Google 日曆。

### Q7: Google Calendar 同步需要網路嗎？

是的，同步至 Google Calendar 時需要網路連線。但系統本身的資料儲存與內建提醒功能不需要網路，可完全離線使用。
